# 学玩融合 — 功能升级设计文档

**日期**：2026-06-09
**状态**：已确认，待实现
**目标**：为「知识PK大作战」添加学习分析和社交分享功能，提升朋友间休闲学习的体验

## 概述

本次升级聚焦两个方向：**强化学习功能**和**增加社交趣味性**，为朋友间休闲学习场景设计。共包含 3 个功能模块，预计 3-5 天完成。

---

## 功能 1：📊 答题报告

### 目标

每局游戏结束后，提供有价值的答题分析，帮助用户了解自己的学习状况。

### 数据展示内容

#### 对战模式报告

- **基础统计**：总题数、双方答对数、正确率
- **速度分析**：平均答题用时、最快一题、最慢一题
- **胜负走势**：折线图展示双方分数变化
- **关键转折**：标注"翻盘点"（连对多题反超的时刻）

#### 单人模式报告

- **基础统计**：正确率、总用时、平均速度
- **速度奖励统计**：获得了多少速度加分
- **知识点分析**：按题目类型/分类的答对率
- **与历史对比**：相比上次是否进步

### 技术实现

```
前端组件：
  GameReport.tsx    - 报告主组件
  ScoreChart.tsx    - 分数走势图（轻量Canvas绘制）
  StatsCard.tsx     - 统计数据卡片

数据来源：
  - 对战模式：已有的 answers[] 和 scores[] 数据
  - 单人模式：已有的答题记录 + localStorage 历史数据
  - 无需后端新增接口，前端计算即可
```

### 交互设计

- 结束页底部新增「查看报告」按钮
- 报告页面支持左右滑动切换不同统计维度
- 底部有「再来一局」和「返回首页」按钮

## 功能 2：📤 结果分享图

### 目标

游戏结束后生成精美的分享图片，用户可以保存或分享给朋友，促进传播和邀请。

### 分享图内容

#### 对战结果分享图

- 标题区：「知识PK大作战」logo + 标题
- 对战区：双方名字 + 比分
- 数据区：正确率对比、最快答题对比
- 高光时刻：翻盘点、极速答题等
- 底部：邀请语 + 链接

#### 单人成绩分享图

- 标题区：「今日答题成绩」
- 数据区：正确率、平均用时、速度加分、总分
- 底部：邀请语 + 链接

### 技术实现

```
方案：使用 html2canvas 将 DOM 元素渲染为图片
  - 零依赖原生方案，无需 Canvas API 手动绘制
  - 预设精美的分享卡片模板
  - 支持保存到本地 + 复制链接

新增依赖：html2canvas

组件：
  ShareCard.tsx       - 分享卡片 DOM 组件
  ShareButton.tsx     - 触发按钮（长按/点击保存）
  shareUtils.ts       - html2canvas 封装工具
```

### 交互设计

- 结束页 / 报告页新增「分享」按钮
- 点击后弹出分享卡片预览，支持「保存图片」和「复制链接」
- 卡片使用渐变背景 + 圆角设计，风格与项目一致

## 功能 3：⭐ 题目收藏/分享

### 目标

让用户收藏好题、难题，创建专属题集，同时支持将题目分享给朋友挑战。

### 收藏功能

- 答题过程中点击 ⭐ 收藏按钮即可收藏当前题目
- 首页新增「我的收藏」入口，展示所有收藏的题目
- 支持按题库分类筛选收藏
- 收藏的题目可以反复练习（类似错题练习的模式）

### 分享功能

- 在收藏列表中选中题目，生成专属「挑战链接」
- 朋友打开链接后进入单人挑战模式，只答这些题
- 分享时附带出题者的成绩作为挑战目标

### 技术实现

```
存储方案：localStorage（与错题系统一致的模式）
  favorites.json: {
    questions: [
      { id, content, options, answer, quizId, addedAt }
    ]
  }

前端组件：
  FavoriteButton.tsx      - 收藏按钮（⭐/☆）
  FavoritesPage.tsx       - 收藏列表页
  FavoritePracticePage.tsx - 收藏题目练习页

路由：
  /favorites           - 收藏列表
  /favorites/practice  - 收藏练习
  /challenge/<base64>  - 分享挑战页面
```

### 交互设计

- 答题卡片右上角显示 ⭐ 收藏按钮
- 收藏列表页支持多选 → 「生成挑战链接」
- 挑战链接页面：展示题目数量和出题者成绩 → 点击开始答题
- 收藏练习复用单人答题流程，只替换题源

## 技术影响总结

### 后端改动

- 无需新增 API 接口
- 无需数据库迁移
- 无需新增 Socket.IO 事件

### 前端改动

- 新增页面：答题报告页、收藏列表页、收藏练习页、挑战页面
- 修改页面：结束页（添加报告/分享按钮）、首页（添加收藏入口）
- 新增组件：GameReport、ScoreChart、StatsCard、ShareCard、ShareButton、FavoriteButton 等
- 新增路由：/report、/favorites、/favorites/practice、/challenge/:data

### 新增依赖

- html2canvas：分享图生成

### 数据存储

- 全部使用 localStorage，与现有错题系统保持一致
- 无需用户账号体系
